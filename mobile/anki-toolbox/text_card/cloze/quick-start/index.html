快速入门

挖空方式​

有两种推荐方案：

  • 使用WPS或Word挖空：在文档中对需要挖空的部分进行加粗、标红等，详见：Word挖空制卡
  • 使用Markdown笔记软件挖空：使用Obsidian、思源笔记、Typora等Markdown笔记软件，利用快捷键设置加粗、斜体等样式进行挖空。

这里以Obsidian为例介绍文本挖空（当然你用其他Markdown软件也是类似）。

首先打开Obsidian，按图示新建一条笔记，然后把要挖空的材料粘贴进正文编辑器内。接着在编辑器中选中要挖空的文本，然后按下Ctrl+B快捷键(Mac上用Cmd+B)，就可以对文本添加加粗样式(对应的Markdown语法是**xx**)。

Obsidian界面
提示

为了方便区分不同卡片，可以在每张卡片开始位置添加1、这种数字编号，方便后面制卡时选择Front字段的匹配模式。

把要挖空的部分标记好后，就可以准备制卡。按下图所示在文件管理器中找到笔记文件位置备用。

在文件管理中打开笔记 在文件管理中找到笔记

开始制卡​

打开PDF Guru Anki，进入文本制卡菜单的挖空题标签页下，将上步文件管理器中的笔记文件拖入到输入文件输入框中，其他参数按下图填写：

参数示例
注意

下面是每个参数的具体说明，

参数说明
目标牌组选择卡片存放的目标牌组，用户可以先在Anki中创建目标牌组，然后在这里选择该牌组
创建子牌组是否根据牌组标识符创建子牌组
制卡模式文本挖空的模式，支持遮一猜一、遮全猜一、遮全猜全、自由猜等
字段映射设置字段的正则匹配模式，一般只需要关注Front和Back的匹配模式，其他可留空
挖空语法需要进行挖空文本语法格式
忽略分组当使用{{c1::xx}}或[[c1::xx]]挖空语法时，是否忽略挖空编号，全部重排编号
一空一卡只对挖空模式为遮一猜一或遮全猜一时有效，表示是否每个挖空单独生成一张卡片
文档类型当前文档的类型，默认为TXT，如果设置为Markdown，会自动对Markdown的语法进行解析
Obsidian语法只有当文档类型为Markdown时有效，开启后支持解析Obsidian的图片语法
媒体文件目录只有当文档类型为Markdown时有效，表示图片等媒体文件所在文件夹
输入文件填写文档的绝代路径，可以将文档拖入自动填充

点击确认按钮后就会开始制卡，稍等片刻制卡成功后，就可以在Anki中查看生成的卡片。
